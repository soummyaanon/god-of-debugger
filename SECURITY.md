# Security policy

## Supported versions

Security-sensitive fixes are applied to the default branch (`main`) and reflected in the current marketplace catalog and plugin version in [`plugins/god-of-debugger/.claude-plugin/plugin.json`](plugins/god-of-debugger/.claude-plugin/plugin.json). There is no long-term support policy for older tags; use the latest release when possible.

## Reporting a vulnerability

**Please do not file a public issue** for undisclosed security problems.

Instead, use **[GitHub Security Advisories](https://github.com/soummyaanon/god-of-debugger/security/advisories/new)** to report vulnerabilities privately. That lets maintainers work on a fix before details are public.

Include:

- A short description of the issue and its impact
- Steps to reproduce (if applicable)
- Affected paths, versions, or configuration (if known)

We aim to acknowledge reports in a reasonable timeframe and coordinate disclosure after a fix is available.

## Scope

This policy covers this repository and the artifacts it ships (marketplace metadata and plugin content). It does not cover third-party services (e.g. hosting or AI providers); report those issues to the respective vendors.
